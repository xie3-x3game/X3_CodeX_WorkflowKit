# 任务分流规则

本文件帮助总控窗口判断新问题应该交给哪个窗口。

## 快速分流表

| 用户意图 | 主窗口 | 辅助窗口 |
| --- | --- | --- |
| 判断 CodeX 是否适合做某类事 | 能力边界 | 提示描述 |
| 改进任务描述、提示词、验收标准 | 提示描述 | 能力边界 |
| 制定策略、比较方案、判断优先级和取舍 | 策略拟定 | 总控 |
| 设置提醒、周期复盘、长期跟进 | 长期任务 | 总控 |
| 拆分多个窗口或多个代理工作 | 多项代理 | 总控 |
| 开发、调试、测试、提交代码 | 软件开发 | 工具插件 |
| 研究 shell、浏览器、GitHub、Skills、插件 | 工具插件 | 软件开发 |
| 剧本、人物、对白、审稿 | 写作剧本 | 提示描述 |
| 学习计划、练习、测验、复盘 | 自我学习 | 资料整理 |
| 外部学习项目经验同步、方法论回收 | 总控 | 自我学习 |
| 摘要、资料整理、知识库结构 | 资料整理 | 提示描述 |
| 更新窗口体系、目录结构、迁移规则 | 总控 | 策略拟定 |

## 超出边界时的处理

当窗口发现任务超出自身边界：

1. 停止深入展开。
2. 用一句话说明超出边界的原因。
3. 指定应转交的目标窗口。
4. 按标准交接格式写出必要上下文。
5. 回到总控窗口，由总控更新目标窗口的任务文件。

## 多窗口任务优先级

- 有明确执行对象时，优先交给执行窗口。
- 有方法论问题时，优先交给研究窗口。
- 有跨窗口协调时，留在总控。
- 有并行拆分时，先交给多项代理设计分工。

## 交接最低要求

转交任务必须包含：

- 原始问题。
- 当前窗口已经判断出的边界。
- 目标窗口需要知道的上下文。
- 期望输出。
- 完成后应该回填到哪个文件。
