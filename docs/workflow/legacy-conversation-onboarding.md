# 混乱对话接入流程

本流程用于把已经进行了一段时间、上下文混乱、边界不清的旧窗口接入多窗口协作体系。

## 适用场景

- 旧窗口已经聊了很多，但没有明确窗口职责。
- 用户在错误窗口提出过需求。
- 有些任务被口头转交后遗忘。
- 新工程已有一批对话，想接入 `docs/windows`、`docs/tasks`、`docs/dispatch` 体系。

## 接入目标

不要试图一次性整理完整历史。先让旧窗口进入可控状态：

1. 明确当前窗口身份。
2. 摘要当前未完成事项。
3. 区分本窗口任务和应转交任务。
4. 把转交任务写入 Dispatch 队列。
5. 后续按新规则运行。

## 旧窗口接入命令

把下面内容发到旧窗口：

```text
这是一个已有的旧窗口，现在要接入项目的多窗口协作体系。

请先读取：
1. AGENTS.md
2. docs/workflow/window-boot-protocol.md
3. docs/workflow/window-preflight-check.md
4. docs/dispatch/README.md

当前窗口名称：<填写窗口名>

请做旧窗口接入，不要继续展开新任务：

1. 判断本窗口应加载的 docs/tasks 和 docs/windows 文件。
2. 用 5-10 条总结当前对话里仍然有效的上下文。
3. 列出当前对话里未完成的任务。
4. 判断每个任务是否属于本窗口边界。
5. 属于本窗口的，整理到本窗口后续任务建议。
6. 不属于本窗口的，按 Dispatch 任务格式生成转交摘要。
7. 最后给出建议：继续本窗口、转交其他窗口、放弃哪些任务。
```

## 输出格式

旧窗口接入后应输出：

```text
已识别窗口：
应读取任务文件：
应读取工作指南：

仍然有效的上下文：
-

本窗口可继续处理的任务：
-

应转交的任务：
- 任务标题：
  目标窗口：
  转交原因：
  必要上下文：
  期望产出：

建议放弃或暂缓的任务：
-

建议下一步：
```

## 总控接收旧窗口摘要

用户把旧窗口输出贴回总控后，总控负责：

1. 确认窗口边界判断是否正确。
2. 为应转交任务创建 `docs/dispatch/queue/T-*.md`。
3. 必要时更新 `docs/tasks/<window>.md`。
4. 如果对应 `docs/topics/*.md` 缺失，先补齐主题记录文件，再让窗口继续普通任务。
5. 如果没有应转交任务，不创建空 dispatch。
6. 运行 dispatch 校验和窗口一致性校验。
7. 告诉用户下一步去哪个窗口执行。

## 重要规则

- 旧窗口接入时不要先让它继续执行任务。
- 先整理，再分流，再执行。
- 不把整段旧对话复制进长期文档，只提炼仍然有效的上下文。
- 不确定的任务标记为 `Inbox` 或 `Blocked`，不要强行归类。
