# 总控窗口工作指南

## 窗口定位

总控窗口负责维护 CodeX 使用研究的全局认知地图。它不深入执行专项任务，而是决定问题应该放在哪个窗口、哪些结论需要沉淀、哪些规则可以迁移到新项目。

## 适合处理的问题

- 新问题应该进入哪个专项窗口。
- 当前窗口体系是否需要调整。
- 专项窗口形成的结论如何合并进总索引。
- 新项目应该生成哪些目录和 Markdown 指南。
- 当前任务应该分配给哪个窗口。
- 阶段性复盘 CodeX 使用方式的变化。

## 不适合处理的问题

- 具体代码开发。
- 长篇资料整理。
- 剧本内容创作。
- 单一工具或插件的深度实验。
- 长时间调试或验证某个具体问题。

## 标准工作流程

1. 判断用户问题属于总控、专项还是临时实验。
2. 如果属于专项，提醒应该切换到哪个窗口。
3. 必要时更新 `docs/tasks/<window>.md`，写清任务概要、上下文和交接要求。
4. 如果属于总控，更新登记表、索引、规则或项目脚手架。
5. 专项窗口回填结论时，提炼为稳定结论、限制条件和后续问题。
6. 定期检查窗口体系是否过多、过少或边界重叠。

## 产出物

- `docs/codex-research-index.md`
- `docs/window-registry.md`
- `docs/windows/*.md`
- `docs/tasks/*.md`
- `docs/workflow/*.md`

## 回填格式

```text
来源窗口：
结论：
证据：
限制条件：
需要更新的文件：
后续问题：
```
