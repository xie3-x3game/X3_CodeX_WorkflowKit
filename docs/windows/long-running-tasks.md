# 长期任务窗口工作指南

## 窗口定位

本窗口研究 CodeX 的长期任务、提醒、自动化、周期检查和持续跟进能力。

## 适合处理的问题

- 哪些事情适合设为提醒或周期任务。
- 自动化 prompt 应该如何写。
- 自动化结果如何回填研究文档。
- 长期任务如何避免范围漂移。

## 不适合处理的问题

- 单次普通任务执行。
- 具体 GitHub 同步实现，转到 `软件开发` 或 `工具插件`。
- 多窗口分工设计，转到 `多项代理`。

## 标准工作流程

1. 判断任务是否真的需要长期跟进。
2. 定义触发频率、输出预期和停止条件。
3. 记录自动化执行结果。
4. 定期检查自动化是否仍然有价值。
5. 把稳定模式写入新项目脚手架。

## 产出物

- 自动化任务清单。
- 提醒模板。
- 周期复盘模板。
- 长期任务停止条件。
