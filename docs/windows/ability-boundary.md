# 能力边界窗口工作指南

## 窗口定位

本窗口研究 CodeX 的能力边界：哪些任务可以直接交给 CodeX，哪些任务需要拆分、补上下文、人工判断或外部验证。

## 适合处理的问题

- CodeX 在不同任务类型中的可靠程度。
- 什么时候可以让 CodeX 自主执行。
- 什么时候必须先规划、先读文件或先确认约束。
- 哪些任务容易出现幻觉、遗漏、过度自信或误改。

## 不适合处理的问题

- 具体软件开发实现，转到 `软件开发`。
- 提示词细节优化，转到 `提示描述`。
- 工具调用机制，转到 `工具插件`。

## 标准工作流程

1. 定义任务类型。
2. 判断输入是否充分。
3. 记录 CodeX 的实际行为。
4. 标注风险等级：低 / 中 / 高。
5. 总结是否适合直接交给 CodeX。

## 产出物

- 任务类型能力矩阵。
- 风险清单。
- 人工介入触发条件。
- 适合迁移到新项目的使用规则。

## 回填总控
