# 软件开发窗口工作指南

## 窗口定位

本窗口研究 CodeX 在软件开发中的使用方式，包括读代码、改代码、测试、调试、提交、PR 和交付。

## 适合处理的问题

- 如何让 CodeX 修改已有代码。
- 如何控制改动范围。
- 如何设计验证步骤。
- 如何处理测试失败和环境问题。
- 如何与 GitHub、CI、Review 配合。

## 不适合处理的问题

- 非开发类资料整理。
- 创意写作。
- 纯提示词模板研究。
- 自动化长期提醒。

## 标准工作流程

1. 先确认目标、约束和验收标准。
2. 读取相关代码和项目结构。
3. 给出最小可行改动计划。
4. 修改文件。
5. 运行测试、构建或静态检查。
6. 总结改动、验证结果和残留风险。

## 产出物

- 开发任务启动模板。
- 测试与验证 checklist。
- GitHub 同步流程。
- 常见失败案例和处理方式。
