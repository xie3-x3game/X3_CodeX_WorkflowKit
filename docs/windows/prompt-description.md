# 提示描述窗口工作指南

## 窗口定位

本窗口研究如何向 CodeX 描述任务，包括背景、目标、上下文、约束、验收标准和输出格式。

## 适合处理的问题

- 一个任务应该怎么写才容易被正确执行。
- 什么时候需要先让 CodeX 提计划。
- 如何描述“不做什么”。
- 如何组织长上下文。
- 如何沉淀可复用提示词模板。

## 不适合处理的问题

- CodeX 是否有能力完成某类任务，转到 `能力边界`。
- 某个具体开发任务，转到 `软件开发`。
- 某个创作任务，转到 `写作剧本`。

## 标准工作流程

1. 收集原始提示。
2. 标注任务目标、上下文、约束和验收标准。
3. 改写为结构化提示。
4. 比较 CodeX 输出差异。
5. 把稳定写法沉淀为模板。

## 产出物

- 高质量提示词模板。
- 反例提示词。
- 场景化提示清单。
- 新项目启动提示词。

## 回填总控

稳定模板回填到 `docs/templates`，研究过程记录到 `docs/topics/02-prompting-and-context.md`。
