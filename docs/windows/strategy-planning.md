# 策略拟定窗口工作指南

## 窗口定位

本窗口研究 CodeX 如何辅助制定策略、比较方案、判断优先级、明确取舍，并把模糊目标转成可执行路线。

## 适合处理的问题

- 如何从目标、约束和资源出发制定策略。
- 如何比较多个方案的收益、风险、成本和机会。
- 如何确定阶段目标、优先级和行动路线。
- 如何建立决策标准、风险清单和验证指标。
- 如何把一次策略讨论沉淀成决策记录或路线图。

## 不适合处理的问题

- 跨窗口体系调整、项目级协调和规则维护，转到 `总控`。
- 具体代码实现、测试和交付，转到 `软件开发`。
- 具体剧本、正文、对白创作，转到 `写作剧本`。
- 学习路径、练习和测验设计，转到 `自我学习`。
- 大量资料摘要和知识库整理，转到 `资料整理`。
- 提示词细化和验收标准表达，转到 `提示描述`。
- 周期跟进和提醒安排，转到 `长期任务`。

## 标准工作流程

1. 明确策略对象、目标、时间尺度和不可突破的约束。
2. 列出关键事实、假设、利益相关方和资源条件。
3. 生成 2-4 个可选方案，并说明各自适用条件。
4. 用统一标准比较方案：收益、成本、风险、可逆性、验证难度。
5. 给出推荐策略、阶段路线、关键取舍和风险预案。
6. 标记需要转交其他窗口执行或验证的事项。

## 产出物

- 策略简报。
- 方案对比表。
- 决策记录。
- 阶段路线图。
- 风险与假设清单。
- 执行前验证指标。
